# 要求内容

## 概要

クイズ画面で単語にスターをつけられるようにする。スター済み単語はランダム取得時に出現しにくくなり、覚えた単語を効率よく除外できる。

## 背景

単語帳が増えてくると、すでに記憶した単語が繰り返し出題されて学習効率が下がる。スター機能により「もう覚えた」単語に印をつけ、新しい単語や苦手な単語を優先的に出題できるようにする。

## 実装対象の機能

### 1. スタートグル

- クイズ画面でスターアイコンのボタンをタップすると、その単語のスター状態をトグルする
- スター状態は D1 の `phrases` テーブルの `starred` カラムに永続化する
- API 経由でスター状態を更新する（`POST /api/v1/star`）

### 2. スター済み単語の出現確率低下

- ランダムフレーズ取得（`POST /api/v1/phrase`）で、スター済みの単語は通常の 1/10 の確率で出現する
- 完全には除外しない（「表示されにくくなる」）

### 3. スター状態の表示

- クイズ画面にスターボタンを表示する
- 現在のフレーズのスター状態を視覚的にフィードバックする（スター済み: 塗りつぶしスター、未スター: 輪郭スター）

## 受け入れ条件

### スタートグル

- [ ] クイズ画面にスターボタンが表示される
- [ ] スターボタンをタップするとスター状態が切り替わる
- [ ] スター状態が D1 に永続化され、ページリロード後も保持される
- [ ] API が `POST /api/v1/star { phraseId, starred }` を受け付ける

### 出現確率低下

- [ ] スター済み単語は未スター単語の約 1/10 の確率で取得される
- [ ] スター済み単語が完全に除外されることはない

### スター状態の表示

- [ ] フレーズ取得時に `starred` フィールドが返される
- [ ] スターボタンは question 状態・answer 状態の両方で常に表示する
- [ ] スター済みの場合は塗りつぶしスターアイコンを表示する
- [ ] 未スターの場合は輪郭スターアイコンを表示する

## 成功指標

- スター機能で繰り返し出現させたくない単語を管理できる

## スコープ外

以下はこのフェーズでは実装しません:

- スター済み単語の一覧画面
- スターのバッチ解除（全解除など）
- 結果画面でのスター状態の表示
