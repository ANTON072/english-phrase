# 英単語帳アプリ

こんなウェブアプリを作りたい。

## Phase.1

- 英単語帳はすでにNotionで作っている。
- そのNotionのデータをCloudflare D1に保存したい。
- Macのターミナルからコマンドを実行するのか、Github Actionsから実行するのか、こだわりはない。簡単なのがいい。
- Drizzle ORMを使いたい。
- モノレポにしておきたい。
- Phase.1ではDBにNotionのデータを保存するまで。
- Notionを今後も更新していく。手動でDBに保存できるようにしておきたい。
- 英単語が1万件あったとして、初回は1万件登録。次に10件追加したら、次は10件の登録。となるようにしたい。毎回全件書き換えはしない。
- Notion側で単語が削除されたとしても無視する。
- エラー時の挙動など細かいところは後で詰める。

## Phase.2

- Cloudflare WorkersでAPIを作る。DBはPhase.1で作ったものを利用する。
- フレームワークはHonoを使う。
- エンドポイントは1つだけ。アクセスするとランダムで1件のデータを抽出して返却する。連続して同じデータを返却しないようにする。でもそれはフロント側でやればよいか？同じデータだったら再度APIを叩くとかで。

## Phase.3

- Webフロントを作る。ボタンを押すと、Phase2のAPIを実行して表示。それの繰り返し。
- スタートボタンと終了ボタンがある。終了ボタンを押すと、今日はいくつテストしたか表示する。
- 最初に英語が出て、ボタンを押すと日本語が表示される。次へボタンを押すと、Phase2のAPIを実行して表示。繰り返し。
- Vite React SPAなのかHonoでフロントも作るのか検討が必要。SPAのほうが拡張性が高いかな。

## 全体的に

- モノレポでやりたい
- Cloudflareスタックで無料で済むようにしたい。
